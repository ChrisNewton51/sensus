Static analysis warning: global_error is a non-volatile static error struct bound to MMIO shadow 0x40001000, written inside USART2_IRQHandler and read from the main task via cJSON_GetErrorPtr(). Without volatile (or C11 _Atomic / critical-section guard), the compiler may cache a stale offset. Remediation: declare global_error as volatile error, or route ISR updates through a lock-free ring buffer and parse only from main context.

Before

void TIM2_IRQHandler(void) {
  if (TIM2->SR & TIM_SR_UIF) {
    TIM2->SR = ~TIM_SR_UIF;
    g_sys_status.motor_rpm =
      encoder_delta * 60 / PPR;
    g_sys_status.fault_flags |=
      check_overcurrent();
  }
}

After

void TIM2_IRQHandler(void) {
  if (TIM2->SR & TIM_SR_UIF) {
    TIM2->SR = ~TIM_SR_UIF;
    motor_status_snapshot_t snap;
    snap = motor_compute_status(
      encoder_delta);
    motor_status_publish_atomic(&snap);
  }
}

Fact-Grounded Analysis: This summary was synthesized locally utilizing an AST JSON token fact-matrix. No source code was sent out over public network channels.

Why this change matters

  • • Direct ISR writes to g_sys_status create race conditions with the main loop reader — atomic publish isolates the mutation vector.
  • • Encapsulating computation in motor_compute_status() reduces cyclomatic complexity inside the interrupt handler.
  • • Safety-critical firmware domains require bounded ISR execution; deferring non-critical fault aggregation prevents priority inversion.
  • • The snapshot pattern enables deterministic replay during post-mortem fault analysis without locking shared globals.
